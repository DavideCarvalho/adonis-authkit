---
title: Console session on other routes
description: Use the AuthKit console account session to protect any route or integrate third-party packages — like the adonis-telescope dashboard.
---

The AuthKit console (`/account/*`, `/auth/admin`) keeps its own account
session. Sometimes you want to reuse that session **outside** the
console: protecting an admin route of your own, or gating a third-party
package's dashboard to signed-in users only.

For that, `@dudousxd/adonis-authkit-server` exports public helpers — use
them instead of reading `ctx.session` with the internal key
(`account_user_id`), which is an implementation detail and may change:

```ts
import {
  getAccountId,       // string | null — id of the signed-in account
  hasAccountSession,  // boolean
  consoleLoginUrl,    // login URL with an optional return_to
} from '@dudousxd/adonis-authkit-server'
```

## Protecting your own route

```ts
router
  .get('/reports', [ReportsController])
  .use(async (ctx, next) => {
    if (!hasAccountSession(ctx)) {
      return ctx.response.redirect(consoleLoginUrl('/reports'))
    }
    return next()
  })
```

After login, `return_to` sends the user back to `/reports`.

## Recipe: protecting adonis-telescope

[`@dudousxd/adonis-telescope`](https://davidecarvalho.github.io/adonis-telescope/)
accepts an `authorize` hook that may return `{ redirectTo }` — a perfect
fit for the console session:

```ts
// config/telescope.ts
import { defineConfig } from '@dudousxd/adonis-telescope'
import { hasAccountSession, consoleLoginUrl } from '@dudousxd/adonis-authkit-server'

export default defineConfig({
  enabled: true,

  /**
   * Humans come in through the console session: with no session, redirect
   * to the AuthKit login and return to /telescope after authenticating.
   */
  authorize: async (ctx) =>
    hasAccountSession(ctx) ? true : { redirectTo: consoleLoginUrl('/telescope') },

  /**
   * Agents (MCP) have no browser session — they use their own Bearer
   * token, separate from the human flow.
   */
  mcp: { token: process.env.TELESCOPE_TOKEN },
})
```

## Restricting by role

`getAccountId()` returns the account id — combine it with your
`accountStore` to check roles before granting access:

```ts
authorize: async (ctx) => {
  const accountId = getAccountId(ctx)
  if (!accountId) return { redirectTo: consoleLoginUrl('/telescope') }

  const store = await ctx.containerResolver.make('authkit.accountStore')
  const account = await store.findById(accountId)
  return account?.role === 'ADMIN'
}
```
