The AuthKit console (`/account/*`, `/auth/admin`) keeps its own account
session. Sometimes you want to reuse that session **outside** the
console: protecting an admin route of your own, or gating a third-party
package's dashboard to signed-in users only.

For that, `@adonis-agora/authkit-server` exports public helpers — use
them instead of reading `ctx.session` with the internal key
(`account_user_id`), which is an implementation detail and may change:

```ts
import {
  getAccountId,       // string | null — id of the signed-in account
  hasAccountSession,  // boolean
  consoleLoginUrl,    // login URL with an optional return_to
} from '@adonis-agora/authkit-server'
```

## Protecting your own route

```ts
router
  .get('/reports', [ReportsController])
  .use(async (ctx, next) => {
    if (!hasAccountSession(ctx)) {
      return ctx.response.redirect(consoleLoginUrl('/reports'))
    }
    return next()
  })
```

After login, `return_to` sends the user back to `/reports`.

## Recipe: protecting adonis-telescope

[`@dudousxd/adonis-telescope`](https://davidecarvalho.github.io/adonis-telescope/)
accepts an `authorize` hook that may return `{ redirectTo }` — a perfect
fit for the console session:

```ts
// config/telescope.ts
import { defineConfig } from '@dudousxd/adonis-telescope'
import { hasAccountSession, consoleLoginUrl } from '@adonis-agora/authkit-server'

export default defineConfig({
  enabled: true,

  /**
   * Humans come in through the console session: with no session, redirect
   * to the AuthKit login and return to /telescope after authenticating.
   */
  authorize: async (ctx) =>
    hasAccountSession(ctx) ? true : { redirectTo: consoleLoginUrl('/telescope') },

  /**
   * Agents (MCP) have no browser session — they use their own Bearer
   * token, separate from the human flow.
   */
  mcp: { token: process.env.TELESCOPE_TOKEN },
})
```

## Restricting by role

`getAccountId()` returns the account id — combine it with your
`accountStore` to check roles before granting access:

```ts
authorize: async (ctx) => {
  const accountId = getAccountId(ctx)
  if (!accountId) return { redirectTo: consoleLoginUrl('/telescope') }

  const store = await ctx.containerResolver.make('authkit.accountStore')
  const account = await store.findById(accountId)
  return account?.role === 'ADMIN'
}
```

## Sudo mode (step-up authentication)

Some operations should require recent re-authentication even when the user already has a
valid session. AuthKit ships a **sudo mode** — after the user confirms their identity (password
or passkey), a timestamp is stored in the session and remains valid for a configurable grace
period (default 15 minutes). The `sudo_mode` runtime setting controls both `enabled` and
`graceMinutes`.

`@adonis-agora/authkit-server` exports the helpers you need in your own controllers:

```ts
import {
  requireSudo,        // redirects/401s when sudo grace has expired
  isSudoActive,       // boolean check (no side-effects)
  markSudo,           // stamp the session after a successful confirmation
  SUDO_SESSION_KEY,   // raw session key ('authkit_sudo_at')
} from '@adonis-agora/authkit-server'
```

### Protecting a sensitive route

```ts
// app/settings/controllers/delete_account_controller.ts
import type { HttpContext } from '@adonisjs/core/http'
import { requireSudo } from '@adonis-agora/authkit-server'

export default class DeleteAccountController {
  async handle(ctx: HttpContext) {
    // Redirects to /account/confirm when sudo grace has expired.
    // Respects the `sudo_mode` runtime setting (enabled + graceMinutes).
    await requireSudo(ctx)

    // ... proceed with deletion
  }
}
```

### Checking without redirecting

```ts
import { isSudoActive } from '@adonis-agora/authkit-server'

const active = isSudoActive(ctx, graceMinutes)
if (!active) {
  return ctx.response.unauthorized({ error: 'sudo_required' })
}
```

### Marking sudo after a confirmation step

If you add a custom confirmation screen, call `markSudo` after verifying the user's
credentials:

```ts
import { markSudo } from '@adonis-agora/authkit-server'

// After verifying password / passkey:
markSudo(ctx) // stamps authkit_sudo_at = Date.now()
```
