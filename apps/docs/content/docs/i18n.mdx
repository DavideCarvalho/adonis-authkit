---
title: Internationalization
description: Translate the host-kit screens — pt-BR by default, zero config.
---

Every user-visible string in the host-kit screens (and the flash / error messages the
controllers produce) lives in a flat catalog of dotted keys. The built-in default is
**pt-BR**, so apps work with **no configuration**. You can override individual keys or
provide whole locales via `i18n`.

## The default catalog

`DEFAULT_MESSAGES` (exported) is the pt-BR catalog — it covers every screen: `login.*`,
`signup.*`, `forgot.*`, `reset.*`, `verify_email.*`, `mfa_challenge.*`, `consent.*`,
`account.*`, `admin.*`, plus the controller errors under `errors.*`. `DEFAULT_LOCALE` is
`'pt-BR'`.

## Configuring

```ts title="config/authkit_server.ts"
import { defineConfig } from '@dudousxd/adonis-authkit-server'
import type { I18nConfig } from '@dudousxd/adonis-authkit-server'

const i18n: I18nConfig = {
  locale: 'en',
  messages: {
    en: {
      'login.title': 'Sign in',
      'login.submit': 'Sign in',
      'login.email_label': 'Email',
      'errors.invalid_credentials': 'Invalid credentials',
      // omitted keys fall back to the pt-BR default
    },
  },
}

export default defineConfig({
  // ...
  i18n,
})
```

| Field | Type | Default | Notes |
| --- | --- | --- | --- |
| `locale` | `string?` | `'pt-BR'` | The active locale. |
| `messages` | `Record<string, Partial<AuthMessages>>?` | — | Extra locales and/or per-key overrides; the active locale is merged **over** the pt-BR default. |

## How resolution works

`resolveMessages(i18n)` merges the active locale's overrides over the pt-BR base. Keys the
chosen locale omits fall back to pt-BR (so partial locales still render fully). The resolved
catalog is exposed on the server config as `messages` (and `locale`), ready for the
renderers.

```ts
import { resolveMessages, translate, DEFAULT_MESSAGES } from '@dudousxd/adonis-authkit-server'

const messages = resolveMessages({ locale: 'en', messages: { en: { 'login.title': 'Sign in' } } })
translate(messages, 'login.greeting', { name: 'Ada' }) // "Olá, Ada" (pt-BR fallback) or your override
```

`translate(messages, key, params?)` returns the string for `key` (falling back to the key
itself when missing) and interpolates `{name}`-style placeholders — leaving unmatched
placeholders intact.
