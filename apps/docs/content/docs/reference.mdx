## Server — `defineConfig` (`@dudousxd/adonis-authkit-server`)

`AuthServerConfigInput`:

| Option | Type | Default | Notes |
| --- | --- | --- | --- |
| `issuer` | `string` | — | Public provider URL; must end with `mountPath`. |
| `adapter` | `AdapterFactory` | — | Storage adapter, e.g. `adapters.database({ connection })`. |
| `clients` | `ClientConfig[]` | — | Registered OIDC clients (see below). |
| `jwks` | `JwksConfig` | — | `{ source: 'managed' \| 'jwks', algorithm?, rotationDays?, keys? }`. |
| `ttl` | `TtlConfig` | see below | Token/session lifetimes; accepts `'15m'`, `'30d'`, or seconds. |
| `globalRolesClaim` | `string` | `'roles'` | Claim name where global roles are emitted. |
| `cookieKeys` | `string[]` | `[]` | Cookie signing keys. |
| `observability` | `ObservabilityConfig` | `{}` | `{ metrics?, jsonRoutes?, dashboard? }`. |
| `accountStore` | `AccountStore` | — | Primary identity contract; derives `findAccount`/`verifyCredentials`. `AccountStore = CoreAccountStore & Partial<Mfa/Webauthn/ProviderIdentity capabilities>`; narrow with `supportsMfa`/`supportsPasskeys`/`supportsProviderIdentity`. See [Account Store](/docs/account-store). |
| `patStore` | `PatStore` | — | Optional; required only for PAT flows. |
| `mountPath` | `string` | `'/oidc'` | Where the host kit mounts the OIDC routes. |
| `render` | `AuthHostRenderer` | — | `inertiaRenderer` / `edgeRenderer`, or omit for headless. |
| `branding` | `BrandingConfig` | — | Per-client theming + first-party list. |
| `social` | `AuthSocialConfig` | — | `{ providers: string[] }`; opt-in social login. |
| `patIntrospectionSecret` | `string` | — | Shared secret authenticating PAT introspection. |
| `rateLimit` | `RateLimitConfigInput` | disabled | Opt-in anti-brute-force throttles. |
| `lockout` | `LockoutConfigInput` | enabled | Progressive per-email lockout. See [Account Lockout](/docs/account-lockout). |
| `mail` | `MailHooks` | — | `onPasswordReset` / `onEmailVerification` hooks. |
| `audit` | `AuditSink` | no-op | Best-effort audit sink. |
| `events` | `EventsConfigInput` | — | `{ onEvent?, webhook? }` — observe every audit event in-process or via an HMAC-signed webhook. See [Events & Webhooks](/docs/events). |
| `mfaIssuer` | `string` | `'AuthKit'` | TOTP issuer label shown in authenticator apps. |
| `webauthn` | `WebauthnConfigInput` | derived from `issuer` | Passkey RP params. See [WebAuthn](/docs/webauthn). |
| `i18n` | `I18nConfig` | English (`en`); pt-BR built in | Host-kit screen translations. See [Internationalization](/docs/i18n). |
| `dynamicRegistration` | `DynamicRegistrationConfigInput` | disabled | RFC 7591/7592. See [Dynamic Registration](/docs/dynamic-registration). |
| `admin` | `AdminConfigInput` | disabled | The `/admin` console. See [Admin Console](/docs/admin-console). |
| `adminApi` | `AdminApiConfigInput` | disabled | The `/api/authkit/v1` REST API (API-key auth). See [Admin REST API](/docs/admin-api). |
| `deviceFlow` | `{ enabled: boolean }` | disabled | Device Authorization Grant (RFC 8628). See [Device Flow](/docs/device-flow). |
| `dpop` | `{ enabled: boolean }` | disabled | DPoP sender-constrained tokens (RFC 9449). See [Security](/docs/security#dpop). |
| `par` | `{ enabled: boolean; requirePushedAuthorizationRequests?: boolean }` | disabled | Pushed Authorization Requests (RFC 9126). See [Security](/docs/security#par). |
| `stepUp` | `{ acrValues?: string[]; mfaAcr?: string }` | `mfaAcr: 'urn:authkit:mfa'` | Step-up auth via `acr_values`. See [Security](/docs/security#step-up-authentication). |
| `trustedDevices` | `{ enabled?: boolean; days?: number }` | enabled, 30 days | Skip MFA on a trusted device via an encrypted cookie. See [MFA](/docs/mfa#trusted-devices). |
| `uploads` | `{ avatars?: { disk?: string; directory?: string; maxSizeMb?: number } }` | app's default drive disk, `authkit/avatars`, 5MB | Avatar upload via the app's `@adonisjs/drive`; degrades to URL input when drive is absent. See [Host Kit](/docs/host-kit#avatar-uploads). |
| `passwordless` | `{ magicLink?: boolean; passkeyFirst?: boolean }` | both disabled | Magic-link email login and/or passkey-first login. See [Passwordless](/docs/passwordless). |
| `login` | `{ requireVerifiedEmail?: boolean }` | `false` | Block logins for unverified emails. Capability-probed. See [Compliance](/docs/compliance). |
| `accessTokens` | `AccessTokensConfig` | `{ format: 'opaque' }` | JWT access tokens (RFC 9068) and per-resource config. See [Security](/docs/security#jwt-access-tokens-rfc-9068). |
| `botProtection` | `BotProtectionConfigInput` | disabled | Pluggable CAPTCHA/challenge for login, signup, reset. See [Security](/docs/security#bot-protection). |
| `notifications` | `{ newLoginEmail?: boolean; newDeviceEmail?: boolean }` | both `true` | Security email notifications. See [Security](/docs/security#new-device-login-notification). |
| `resolveGeo` | `(ip: string) => Promise<string \| null>` | — | Pluggable IP → location lookup for session context. See [Admin Console](/docs/admin-console#session-context). |
| `organizations` | `OrganizationsConfigInput` | auto | Multi-tenancy. See [Organizations](/docs/organizations). |

### `ttl` defaults

| Field | Default |
| --- | --- |
| `accessToken` | `900` (15m) |
| `refreshToken` | `2592000` (30d) |
| `idToken` | `900` (15m) |
| `session` | `604800` (7d) |

### `ClientConfig`

| Field | Type | Notes |
| --- | --- | --- |
| `clientId` | `string` | — |
| `clientSecret` | `string?` | Omit for public clients. |
| `redirectUris` | `string[]` | Allowed authorization redirect URIs. |
| `postLogoutRedirectUris` | `string[]?` | For RP-initiated logout. |
| `grants` | `string[]?` | e.g. `authorization_code`, `refresh_token`, `urn:ietf:params:oauth:grant-type:token-exchange`. |
| `tokenEndpointAuthMethod` | `'client_secret_basic' \| 'client_secret_post' \| 'none'` | `'none'` for public/SPA + PKCE. |

### `RateLimitConfigInput`

| Field | Type | Default |
| --- | --- | --- |
| `enabled` | `boolean` | `false` |
| `login` | `{ points, duration }` | `{ points: 10, duration: '1 min' }` |
| `introspection` | `{ points, duration }` | `{ points: 60, duration: '1 min' }` |
| `store` | `string?` | host default store |

## Client — `defineConfig` (`@dudousxd/adonis-authkit-client`)

`ClientConfigInput`:

| Option | Type | Default | Notes |
| --- | --- | --- | --- |
| `issuer` | `string` | — | IdP issuer URL. |
| `clientId` | `string` | — | This client's id. |
| `clientSecret` | `string?` | — | For confidential clients. |
| `redirectUri` | `string` | — | This client's callback URI. |
| `resolver` | `ResolverFactory` | — | `resolvers.jwt(...)` or `resolvers.pat(...)`. |
| `resolveUser` | `(identity, ctx) => Promise<unknown>` | — | Map identity to your user. |
| `resolveAppRoles` | `(identity) => Promise<string[]>` | — | App-local roles. |
| `sessionKey` | `string` | `'authkit'` | Session key for the token set. |
| `scopes` | `string[]` | `['openid','profile','email','offline_access']` | Requested scopes. |
| `globalRolesClaim` | `string` | `'roles'` | Claim to read global roles from. |
| `onBackchannelLogout` | `BackchannelLogoutCallback` | — | Invoked with `{ sid, sub }` on a valid logout token. See [Back-Channel Logout](/docs/back-channel-logout). |
| `sessionIndex` | `SessionIndex` | — | Maps OP `sid`/`sub` to local session ids for back-channel logout. |

### Resolver factories

| Factory | Config | Notes |
| --- | --- | --- |
| `resolvers.jwt` | `{ tokenSource?, jwksUri? }` | Local JWKS validation; `tokenSource` defaults to `'session'`, `jwksUri` to `${issuer}/jwks`. |
| `resolvers.pat` | `{ introspectionUrl, introspectionSecret }` | Validates PATs via the IdP introspection endpoint. |
| `resolvers.opaque` | `{ tokenSource?, introspectionUrl?, cacheTtlMs? }` | Per-request introspection of opaque access tokens (immediate revocation). Requires a confidential client. |

### Login helpers

| Export | Purpose |
| --- | --- |
| `generatePkce()` | Build a PKCE `{ verifier, challenge, method: 'S256' }`. |
| `buildAuthorizeUrl(params)` | `${issuer}/auth` authorization URL. |
| `exchangeCode(params)` | Swap an authorization code for a `TokenSet`. |
| `refreshTokens(params)` | `grant_type=refresh_token` refresh (handles rotation). See [Refresh Tokens](/docs/refresh-tokens). |
| `exchangeToken(params)` | RFC 8693 token-exchange (impersonation). |
| `buildEndSessionUrl(params)` | `${issuer}/session/end` RP-initiated logout URL. |

## SDK — `createAuthkit` (`@dudousxd/adonis-authkit-sdk`)

`createAuthkit(options): Promise<Authkit>` builds the [backend SDK](/docs/sdk).

| Export | Purpose |
| --- | --- |
| `createAuthkit({ mode: 'remote', baseUrl, apiKey, fetchImpl? })` | HTTP driver over the [Admin REST API](/docs/admin-api). |
| `createAuthkit({ mode: 'embedded', app })` | In-process driver resolving server services from the AdonisJS container. |
| `AuthkitApiError` | Error thrown by the remote driver (`status`, `code`, `message`). |

See the [SDK guide](/docs/sdk) for the full method table.

## Ace commands

Commands provided by `@dudousxd/adonis-authkit-server`:

| Command | Notes |
| --- | --- |
| `node ace authkit:doctor` | Validate the config and peer dependencies; emits findings with `ok`/`warn`/`error` levels. |
| `node ace authkit:eject` | Copy the bundled host-kit controllers/pages into your project for customization. |
| `node ace authkit:users:import --file <path> [--dry-run]` | Bulk-import users from a JSON array or NDJSON file, preserving hashes as-is for lazy rehash on login. |
| `node ace authkit:keys:rotate [--keep=N] [--retire] [--dry-run]` | Rotate signing keys in a managed keystore file; keeps a grace-period window of N previous keys (default 2). |

### `authkit:users:import`

See [Passwords & Migration](/docs/passwords-and-migration#user-import-command) for the file format and field list.

### `authkit:keys:rotate`

See [Security — Signing key rotation](/docs/security#signing-key-rotation) for the flag reference and rotation strategy.

## Client helpers — `@dudousxd/adonis-authkit-client`

Additional exports beyond the resolver factories:

| Export | Purpose |
| --- | --- |
| `verifyJwtAccessToken(token, options)` | Verify a JWT AT (RFC 9068) locally against the IdP's JWKS. See [Client](/docs/client#jwt-access-token-verification). |
